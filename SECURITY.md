# Security Policy

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in any CC3PO project, please report it privately.

**DO NOT** create a public GitHub issue for security vulnerabilities.

### How to Report

1. Email: **legal@cc3po.com**
2. Include a detailed description of the vulnerability
3. Include steps to reproduce (if applicable)
4. Include the affected repository and version

### What to Expect

- **Acknowledgment** within 48 hours
- **Initial assessment** within 5 business days
- **Fix timeline** communicated after assessment

## Scope

This security policy covers all repositories under the carloscbrls and cc3po GitHub organizations.

## Responsible Disclosure

We ask that you:
- Allow us reasonable time to fix the issue before disclosure
- Do not exploit the vulnerability
- Act in good faith to help protect our users and systems

Thank you for helping keep CC3PO secure.
